## 2. 技术栈说明

-   **核心框架**: Vue.js 3 (通过 CDN 引入)。选择 Vue 是因为它易于上手，并且其响应式系统和组件化思想非常适合构建单页面应用，即使在单个 HTML 文件中也能保持代码的结构化。
-   **UI 样式**: Tailwind CSS (通过 CDN 引入)。提供原子类，能快速构建现代化、响应式的界面，并轻松实现暗色模式。
-   **图标**: Heroicons。提供一套美观、一致的 SVG 图标。
-   **开发模式**: 所有代码（HTML, CSS, JavaScript）都将集成在单个 `index.html` 文件中，无构建步骤，符合用户要求。
-   **状态管理**: 使用一个全局的 Vue 响应式对象来管理应用状态（如用户登录状态、当前页面、语言等），并利用 `localStorage` 进行数据持久化，以模拟真实的用户会话和设置保存。

## 3. 路由定义

由于是单文件应用且无路由库，将通过一个 `currentPage` 状态变量来控制显示哪个页面/组件。这是一种简单的前端路由模拟。

| `currentPage` 值 | 对应的页面/视图 |
|------------------|-------------------|
| `auth`           | 登录/注册页面     |
| `tool`           | 核心工具页面      |
| `pricing`        | 定价/充值页面     |
| `account`        | 个人中心页面      |

## 4. API 定义

该项目为纯前端应用，所有功能（如用户认证、AI 内容生成、支付）均为前端模拟，不涉及后端 API 调用。

## 5. 数据模型

应用数据将存储在浏览器的 `localStorage` 中，以 JSON 字符串形式保存。主要数据结构如下：

```typescript
interface AppData {
  user: {
    email: string | null;
    isLoggedIn: boolean;
    credits: number; // 剩余免费次数
    plan: 'Free' | 'Pro' | 'Enterprise'; // 当前套餐
    isFirstLogin: boolean; // 是否首次登录，用于新手引导
  } | null;
  settings: {
    language: 'en' | 'zh' | 'ja'; // 当前语言
    darkMode: boolean; // 是否为暗色模式
    emailNotifications: boolean;
    autoSaveHistory: boolean;
  };
  history: {
    date: string;
    productName: string;
    language: string;
    platform: string;
  }[];
}
```

## 6. 主要组件划分

尽管所有代码都在一个文件中，但在逻辑上会划分为以下 Vue 组件：

-   **Root Component**: 根组件，管理全局状态和页面切换。
-   **AuthPage**: 包含登录和注册表单。
-   **ToolPage**: 核心功能，包括输入表单和输出结果展示。
-   **PricingPage**: 定价方案展示。
-   **AccountPage**: 用户个人中心。
-   **Modal**: 可重用的模态框组件，用于显示新手引导、升级提示、支付表单等。
-   **Header**: 网站的导航栏。
-   **Footer**: 网站的页脚。
